Design... como fazemos?

Para obter melhor organização é bom montar as etapas para não se perder ou cometer um erro bobo. É requerido a qualquer processo seja no back, no front ou na interface de design saber onde se encontra cada coisa da aplicação.

1. O primeiro é o briefing, que é onde faz todo levantamento com o cliente sobre a empresa.
2. O wireframe é usado para fazer um “esqueleto” da interface da aplicação para ter um ponto x para chegar.
3. Estabelecer um cronograma para a distribuição de tarefas (tem disponível em outro arquivo chamado softwares uteis).
4. Ultilizar de bagagem visual para se inspirar (tem disponível em outro arquivo chamado softwares uteis).
5. Desenvolvimento do layout, onde irá ocorrer a projeção/desenho da aplicação.
6. Após desenvolver o layout iremos deixar navegável o design do projeto, para prototipar para o cliente, nunca prototipar por fotos, sempre mostrar a funcionalidade e a responsividade.
7. Agora é hora de desenvolver o front e o back end de acordo com o que é pedido pelo projeto.
8. Usar o site para ver se está tudo certo e fazer ajustes.
9. Hospedar em algum servidor que será bancado por min ou pelo cliente ou ambos.
10. Futuramente este site pode haver manutenção que terá um custo, geralmente e cobrado por hora, entre 60-90 reais.
